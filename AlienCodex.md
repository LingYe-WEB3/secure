## 3. 漏洞分析
```
更新owner状态变量为玩家地址
	在合约中，动态数组可以访问到任何位置，所以我们在了解动态数组的写入位置后可以覆盖到owner的值。
```

## 4. 攻击方法
```
用length--让数组声明所有槽位索引
	在符合i+初始位置等于最大值的地方写入player；
	如何不打开数组索引，那么我们修改元素会失败。
```
